Moral Money Community System – Operational Model

The Moral Money ecosystem is organized around projects.

Projects are the fundamental unit of activity inside the community. Any productive action that contributes to the community can be represented as a project.

Examples include building houses, farming land, installing infrastructure, repairing equipment, providing services or contributing knowledge.

Each project contains several core elements. A project has a creator, a set of participants, a domain of activity and an expected outcome.

Projects follow a simple lifecycle. A project is created, participants join, work is executed and finally the project is closed.

Participants confirm their participation and completion through the system interface.

The system is designed so that the user experience remains simple. Members mainly perform three actions: create projects, join projects and confirm project completion.

Complexity is handled internally by the system rather than by the users.
